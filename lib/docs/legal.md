### 3. User Account Types

BusIn supports three distinct user roles:

#### 3.1 Student Accounts

**Primary Purpose**: Managing personal bus subscriptions

**Capabilities**:
- Submit subscription requests for current semester
- View subscription status (Pending, Approved, Rejected)
- Access personal QR code for bus boarding
- View scannings history
- Manage account information

**Limitations**:
- Cannot approve own subscriptions
- Limited to one active subscription per semester
- Must maintain valid student status

#### 3.2 Staff Accounts

**Primary Purpose**: Operational bus access verification

**Capabilities**:
- Scan student QR codes
- View scan history
- Access basic student verification information

**Approval Required**: Staff accounts require administrative approval before activation.

#### 3.3 Administrative Accounts

**Primary Purpose**: System management and oversight

**Capabilities**:
- All staff capabilities, plus:
- Approve/reject student subscriptions
- Manage bus stops (create, edit, delete)
- Configure semester schedules
- Access analytics dashboard
- View all user information
- Manage user accounts
